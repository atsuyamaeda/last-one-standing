# Last One Standing — プロジェクトビジョン

## 背景

AIの急速な進化、とりわけClaude Codeのようなコーディングエージェントの登場により、ホワイトカラーの業務領域が大きく変容しつつある。SNS上でも「どの仕事が残るのか」という議論が活発化しており、個人としても"最後まで立っている者（Last One Standing）"であるために、自らのスキルと市場価値を再定義する必要がある。

## スコープ

本プロジェクトが対象とするのは、**採用コンサルティング（RPO: Recruitment Process Outsourcing）** の領域である。

プロダクト開発など他の活動も行っているが、RPO業務は安定的なキャッシュフローを生み出すセーフティーネットとして機能しており、ここに対してAI時代の戦略を明確にすることが最優先である。

本プロジェクトで明らかにすること:

1. **何が自動化されるのか／できないのか** — RPO業務を分解し、自動化の可否を判定する
2. **何をポートフォリオにできるのか** — 自動化したシステムをオーケストレーターとしての実力証明にする
3. **何を強みとして伸ばすべきか** — 自動化できない領域を特定し、磨き上げる

## プロジェクトの狙い

| # | 狙い | 概要 |
|---|------|------|
| 1 | RPO業務の言語化 | 採用コンサルティングにおけるスキル・業務・提供サービスを網羅的に棚卸しし、明文化する |
| 2 | 自動化による市場価値の向上 | RPO業務を積極的に自動化し、AIオーケストレーターとしての実力を証明する |
| 3 | 尖ったスキルの特定と強化 | RPO領域でAIに代替できない部分を見極め、そこに集中投資する |

## 要件

### R1. RPO業務・提供サービスの網羅的な言語化

採用コンサルティングにおける自身のスキル・経験・提供可能なサービスを洗い出し、構造化して記述する。これが以降の全工程の起点となる。

### R2. 継続的・低コストな提供システムの構築

言語化したサービスを、再現性のある仕組み（スクリプト・ワークフロー・ツール）として実装し、最小限の人的コストで継続提供できる状態を作る。

### R3. 自動化可否によるスキルの判定

各スキル・業務に対して「自動化できるか／できないか」を判定する。

- **自動化できるもの** → システム化し、ポートフォリオとして保有（R4）
- **自動化できないもの** → AIに置き換えられない強みとして磨き上げる（R5）

### R4. 再現性あるシステムのポートフォリオ化

自動化に成功したシステムを、実績・能力の証明として蓄積する。これ自体がオーケストレーターとしての市場価値を裏付ける。

### R5. 非自動化スキルの先鋭化

自動化できなかったスキルこそが「最後に残る価値」である。これを明確に言語化し、クライアントへの訴求軸として磨き上げる。

## 戦略の全体像

```
RPO業務の棚卸し（R1）
        │
        ▼
  ┌───────────┐
  │ 自動化判定 │（R3）
  └─────┬─────┘
        │
   ┌────┴────┐
   ▼         ▼
 自動化可   自動化不可
   │         │
   ▼         ▼
 システム化  スキル先鋭化
 （R2/R4）   （R5）
   │         │
   ▼         ▼
 ポートフォリオ   差別化の武器
 ＝オーケストレーター  ＝Last One Standing
   としての証明        としての強み
```

## まとめ

RPO・採用コンサルティングという軸足を明確にした上で、AIを使って業務を汎用化・自動化する力を手に入れると同時に、自動化できない領域をさらに尖らせていく。**両輪を回すこと**が本プロジェクトの本質である。
