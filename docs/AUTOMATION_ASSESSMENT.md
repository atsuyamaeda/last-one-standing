# RPO業務 自動化判定（R3）

> 本ドキュメントは、[INTERVIEW_LOG.md](./INTERVIEW_LOG.md) をもとに各業務の自動化可否を判定し、
> ポートフォリオ化（R4）と強み先鋭化（R5）の振り分けを行うものである。

## 判定基準

各業務を以下の3段階で判定する:

| 判定 | 定義 |
|------|------|
| **自動化可** | AIやシステムで完全に代替可能。人間の介入は不要またはレビューのみ |
| **部分自動化** | プロセスの一部をAI/システムで効率化できるが、人間の判断・実行が不可欠な工程が残る |
| **自動化不可** | 対人関係・高度な判断・創造性が核であり、AI代替の余地が極めて小さい |

## 前提

- 4.1（EVP・採用計画策定）と 4.4（採用マーケティング）は実態として一体のプロセスであるため、統合して判定する
- 4.6（オンボーディング）/ 4.8（スクラム採用）/ 4.9（メンタリング）はサービスとして未確立のため、判定対象外とする

## 詳細判定

### 1. EVP・採用計画策定 / 採用マーケティング → 部分自動化

#### 自動化できる部分（システム化候補）

| タスク | 自動化手法 | 期待効果 |
|--------|-----------|---------|
| 市場・競合リサーチ | AIによるデスクトップリサーチの自動化（企業情報収集、求人動向分析） | リサーチ時間の大幅短縮 |
| 採用デックのドラフト生成 | 戦略的方向性をインプットすれば、AI がドラフトを生成。Who/What/Howフレームワークをテンプレ化 | 制作工程の短縮 |
| 求人票の初稿作成 | EVP・ペルソナ情報をもとにAIが初稿生成 | ダイレクトリクルーティング Step 4 と連動 |
| 採用計画のシミュレーション | 採用目標×チャネル別歩留まり率でパイプライン予測をシステム化 | 計画策定の精度・速度向上 |

#### 自動化できない部分（人間の価値）

| タスク | なぜ自動化できないか |
|--------|-------------------|
| 経営陣との対話・壁打ち | 経営者の言語化されていないビジョンや迷いを引き出す能力。信頼関係が前提 |
| 「なぜやるのか」の整理 | 事業の文脈、創業者の想い、市場ポジションを統合する高度な抽象化・構造化 |
| ストーリーの設計判断 | 候補者のペインに刺さるストーリーラインの設計は、市場感覚と経験が不可欠 |
| 期待値コントロール | 過度な訴求による入社後ミスマッチを防ぐ「引き算」の判断 |

### 2. ダイレクトリクルーティング → 部分自動化

**6ステップの自動化マッピング:**

```
Step 1. 要件定義・ペルソナ策定    → 自動化不可（経営対話＋市場判断）
Step 2. インサイト分析・言語化    → 部分自動化（リサーチはAI可、ペインの言語化は人間）
Step 3. 自社アセット整理・EVP構築  → 部分自動化（棚卸しはフレームワーク化可、マッチング判断は人間）
Step 4. 求人票の作成             → 部分自動化（AI初稿生成可、ストーリー設計・最終品質は人間）
Step 5. AI活用スカウト運用       → ★既に自動化進行中（Gemini 3 Pro / 約10秒で文面生成）
Step 6. DB検索軸の最適化         → 部分自動化（検索ロジックの構築・テスト自動化は可能）
```

#### 自動化できる部分（システム化候補）

| タスク | 自動化手法 | 期待効果 |
|--------|-----------|---------|
| スカウト文面生成（Step 5） | **既に実装中**: 求人票×職務経歴書→Gemini 3 Proで10秒生成 | 文面作成工数をほぼゼロに |
| HITL改善ループ（Step 5） | 返信率データを自動フィードバックしプロンプトを最適化 | 文面品質の継続的向上 |
| 候補者DB検索（Step 6） | 検索クエリのテンプレ化・A/Bテスト自動化 | 母集団形成の効率化 |
| 候補者情報のATS自動登録 | **既に実装済み**: HERP未連携媒体→HERP自動登録 | 1日1時間→0分、エラー根絶 |
| スカウト通数のデイリー取得 | **開発中**: 自動集計 | 運用状況の可視化 |
| 媒体横断の候補者重複チェック | 複数媒体の候補者を名寄せし、重複スカウトを防止 | 候補者体験の向上・無駄削減 |

#### 自動化できない部分（人間の価値）

| タスク | なぜ自動化できないか |
|--------|-------------------|
| ペルソナ策定（Step 1） | 経営方針×市場状況×採用実現性の三者を統合する判断 |
| ペインの深掘り・言語化（Step 2） | 「3つ以上の深いペイン」を特定するにはターゲット層への共感と洞察が必要 |
| EVPと課題のマッチング（Step 3） | 「何を言うか」だけでなく「何を言わないか」（期待値コントロール）が重要 |
| 求人票のストーリー設計（Step 4） | ペルソナの課題解決を主軸とした構成設計は、マーケティング的センスが必要 |
| スカウト文面の最終品質判断 | AIが生成した文面が「刺さるか」のHITL判断 |

| 面接リマインダー・日程調整 | カレンダー連携による自動化 | 事務工数の削減 |

#### 自動化できない部分（人間の価値）

|--------|-------------------|

| 訴求内容の整理 | ポジションごとの訴求ポイントの設計はマーケティング的思考が必要 |

| タスク | 自動化手法 | 期待効果 |
|--------|-----------|---------|

### 5. 面接・候補者クロージング → 自動化不可

この業務は、前田氏のRPOサービスにおける**最も代替不可能な価値**である。

#### なぜ自動化できないか

| 要素 | 詳細 |
|------|------|
| **面接設計力** | 見極め×アトラクトを統合したストーリーアウトラインの作成。候補者ごとにカスタマイズが必要 |
| **キャリア対話力** | 「市場価値の3要素（伸びている市場・伸びている企業・打席とスペース）」という独自フレームワークによる対話。候補者のリアルタイムな反応に応じた切り替えが必要 |
| **クロージング技術** | 前提作り→軸の深掘り（抽象⇔具体の往復）→最終アトラクトという精緻なプロセス。候補者の感情・迷い・本音を読み取る力が不可欠 |
| **信頼構築** | 「自社に寄せない」スタンスで候補者の信頼を得る。この信頼がクロージングの基盤 |
| **経営者の代弁** | 代表の考えを深く理解し、自分の言葉で語れる能力。経営経験に裏打ちされた説得力 |

#### 自動化できるのは「準備」の一部のみ

| タスク | 自動化手法 |
|--------|-----------|
| 候補者の経歴リサーチ | AIによる経歴要約・SNS情報収集 |
| 過去の面接記録の要約 | ATS上の面接メモの自動要約 |
| 質問リストのドラフト | ポジション要件から質問候補を自動生成（ただし最終判断は人間） |

### 6. 採用基盤構築 → 部分自動化

#### 自動化できる部分（システム化候補）

| タスク | 自動化手法 | 期待効果 |
|--------|-----------|---------|
| ATS初期セットアップ手順のテンプレ化 | HERPの初期設定チェックリスト・設定テンプレの標準化 | 新規クライアント対応の高速化 |
| 媒体統合の手順化 | 媒体一覧テンプレ、ログイン情報管理シート、データ移行スクリプト | 23媒体統合のような作業の再現性向上 |
| ダッシュボード構築のテンプレ化 | Looker Studioの採用ファネルダッシュボードテンプレ | クライアントごとの構築時間短縮 |
| 連携自動化の横展開 | HERP未連携媒体→HERP自動登録システムの汎用化 | 新規クライアントへの即時展開 |

#### 自動化できない部分（人間の価値）

| タスク | なぜ自動化できないか |
|--------|-------------------|
| クライアントの現状診断 | 「何がないか」「何が必要か」の判断はヒアリングと経験が必要 |
| ツール選定 | ATS、媒体、分析ツールの組み合わせは企業の規模・予算・フェーズに依存 |
| 運用設計 | 誰が何をどの頻度で行うかの設計はクライアントの組織に依存 |

### 7. 採用業務の自動化開発 → メタレイヤー

この業務は他の業務を自動化する**メタ能力**であり、それ自体がオーケストレーターとしてのポートフォリオとなる。

**既に構築済み / 開発中のシステム:**
- ✅ HERP未連携媒体→HERP候補者自動登録システム
- 🔧 スカウト通数のデイリー自動取得
- 🔧 スカウト文面作成ツール（Gemini 3 Pro活用）

### R5: 磨き上げるべき強み（Last One Standingの武器）

| # | 強み | なぜAIに代替できないか | 訴求ポイント |
|---|------|---------------------|------------|
| 1 | **面接・候補者クロージング** | リアルタイムの対話力、感情の読み取り、信頼構築。「市場価値の3要素」という独自フレームワーク | CxOクラスの採用を一貫してクローズできる |
| 2 | **経営者との対話・EVP構築** | 言語化されていないビジョンを引き出し構造化する力。経営経験に裏打ちされた共感力 | 経営者目線で「なぜやるのか」を共に言語化できる |
| 3 | **エージェントのファンづくり** | 情熱・信頼・人間的魅力でエージェントを動かす。ヘッドハンター経験に基づく業界理解 | エージェント側の行動原理を知り尽くした上でのリレーション構築 |
