# 年間売上1億円のヘッドハンターが、自分の仕事をAIで自動化してみた。残ったのは何だったのか。

## はじめに — なぜ自分の仕事を解体しようと思ったのか

「その仕事、AIでよくない？」

SNSを開けば、毎日のようにこの手の議論が流れてくる。Claude Code、ChatGPT、Gemini。生成AIの進化は凄まじく、ホワイトカラーの仕事が根こそぎ持っていかれるのではないかという不安は、もはや杞憂とは言い切れないフェーズに入った。

私はフォースタートアップス株式会社でヘッドハンターとして年間売上1億円超を記録し、dodaX「HeadHunter of the Year 2023」MVPを受賞した。その後独立し、現在はスタートアップ企業の採用コンサルティング（RPO）を行っている。

正直に言う。**自分の仕事もAIに食われる可能性がある。** そう思った。

だから、自分で自分の仕事を解体することにした。

何が自動化できて、何ができないのか。自分の手で確かめる。それが最も誠実なアプローチだと考えた。本記事は、その過程と結論を共有するものだ。

---

## 前提 — RPO（採用コンサルティング）とは何をする仕事か

まず、私が提供しているサービスの全体像を示す。

- **EVP・採用計画策定** — 経営者と対話し、「この会社はなぜ採用するのか」「どんな人に来てほしいのか」を言語化する
- **ダイレクトリクルーティング** — スカウト媒体を使い、候補者に直接アプローチする
- **選考体験設計** — 面接フローの設計、選考スピードの管理、候補者フォロー
- **採用マーケティング** — 採用デックやWebサイトを通じて、転職市場へのメッセージを設計する
- **エージェントコントロール** — 人材紹介会社との関係構築・管理
- **面接・候補者クロージング** — 自ら面接に出て候補者を口説き、採用を決める

これらを、ひとつひとつ「自動化できるか？」と問い直していった。

---

## 方法 — 4つの軸で業務を解剖する

各業務に対して、以下の4つの軸で自動化の可否を判定した。

1. **入出力の構造化度** — データとして扱えるか、それとも文脈依存か
2. **判断の再現性** — ルール化できるか、暗黙知に依存するか
3. **対人依存度** — 人と対面しなくても完結するか
4. **技術的実現性** — 既存のAI・ツールで実現できるか

4つすべてが「自動化寄り」なら自動化可能。混在していれば部分自動化。3つ以上が「人間寄り」なら自動化不可。

このフレームワークで、自分の業務を一つずつ裁いていった。

---

## 自動化できたもの — 「作業」は確実にAIの領域になる

### 1. 候補者情報の自動登録

**Before:** 毎日30人分の候補者情報を、各媒体からATSに手動で転記。1日1時間。ヒューマンエラーも頻発。

**After:** Claude Codeで自動登録システムを構築。ヒューマンエラーはゼロに。1日1時間が消えた。

これは典型的な「入力が構造的で、判断が不要で、対人性もない」タスク。自動化しない理由がない。

### 2. スカウト文面のAI生成

求人票と候補者の職務経歴書を掛け合わせ、Gemini 3 Proで個別最適化されたスカウト文面を約10秒で生成する仕組みを構築した。

ただし、ここには重要な注意点がある。

**AIが生成した文面をそのまま送ることはしない。** 必ず人間がレビューし、「この文面は本当に刺さるか？」を判断する。この人間によるフィードバック（HITL: Human-in-the-Loop）を通じてプロンプトの精度を上げ、運用フローとして確立していく。

つまり、**スカウト文面の「生成」は自動化できたが、「品質判断」は人間に残る。**

### 3. 採用ダッシュボードと選考スピード監視

Looker Studioで採用ファネルの分析ダッシュボードを構築し、データドリブンでPDCAを回す体制を作った。選考が滞留している候補者を自動で検知する仕組みも実装可能だ。

数字の集計・可視化・アラートは、AIどころか従来のBIツールでもできる領域。ここに人間が時間を使う理由はない。

---

## 部分的に自動化できたもの — 「型」はシステムに、「判断」は人間に

### 4. ダイレクトリクルーティングのプロセス

私のダイレクトリクルーティングは、以下の6ステップで体系化している。

1. **要件定義とペルソナ策定** — 経営層との対話を通じ、事業計画と転職市場を統合
2. **インサイト分析と言語化** — ターゲット層のペイン（潜在的な課題）を3つ以上言語化
3. **自社アセット整理とEVP構築** — 自社の提供価値をペルソナの課題解決に紐づける
4. **求人票の作成** — ストーリーテリングと可読性を両立した求人票
5. **AIスカウト運用** — 求人票×候補者経歴でハイパーソナライズ文面を自動生成
6. **データベース検索軸の最適化** — ターゲット人材を網羅的に抽出する検索ロジック

このうち、**Step 5と6は自動化が進んでいる。** Step 4（求人票）もAIに初稿を書かせることができる。

しかし、**Step 1〜3は自動化できない。**

なぜか。経営者との対話から「なぜ採用するのか」を引き出し、転職市場の実態と突き合わせ、「この会社に来るべき人はどんなペインを抱えているか」を言語化する。これは、経営経験と市場感覚と共感力の掛け算でしか成立しない仕事だ。

**AIは「書く」ことはできる。しかし「何を書くべきか」を決めることはできない。**

### 5. エージェントコントロール

求人票の共有、進捗レポートの生成、パフォーマンスの可視化。これらの事務的な管理業務は自動化できる。

しかし、エージェントコントロールの本質はそこにはない。

**一番大事なのは、エージェントに「この会社のファンになってもらう」ことだ。**

私は前職でヘッドハンター側にいたからわかる。エージェントは、自分が好きな会社に候補者を推薦したい。フィーが50万円高いから優先する、ということはほとんどない。それよりも、「この会社は伸びる。応援したい」と心から思えるかどうかが、推薦の質と量を決める。

だから私は、会社に出向いて勉強会を開いたり、ビジョンを自分の言葉で語ったりする。この「ファンづくり」は、AIにはできない。

---

## 自動化できなかったもの — ここに「最後に残る仕事」がある

### 6. 面接・候補者クロージング

結論から言う。**これが、私の仕事の中で最もAIに代替できない領域だった。**

面接の準備段階から説明する。

まず、採用背景と要件を整理し、「この人のどこを見極めるか」を言語化する。同時に、候補者のニーズを分析し、刺さるアトラクトメッセージを設計する。見極めとアトラクトの双方を統合した「ストーリーのアウトライン」を組む。ここまでが面接前。

面接では、私は「市場価値の定義」について話すことが多い。

候補者に伝えるのはこういうことだ。資本市場では、企業は「この人を採用して利益が上がるか」を最も重視する。だから、定量的な実績が必要になる。そして、実績を上げるには3つの条件がある。

1. **伸びている市場にいること。** 市場全体の成長率が高ければ、成果を出しやすい。
2. **その中で伸びている企業にいること。** 市場が伸びていても、パイを取れていない企業では恩恵を感じられない。
3. **打席に立てる余白があること。** 人員が飽和し業務が細分化された環境では、「これは自分の成果だ」と言える実績が作れない。

こうしたキャリアの本質について対話しながら、同時にその人の人間性を見る。

クロージングでは、まず前提を作る。「あなたにとっても私にとっても、良いマッチングでないと誰も幸せにならない。だから、ありのままの会話をしたい。」このスタンスを明示することで、候補者の心理的な壁を下げ、私のメッセージが届きやすくなる。

その上で、候補者が大切にしている「軸」を、抽象と具体の往復で深掘りする。何を大事にしているかを聞き、それを具体化し、これまでの選考で感じたことと照らし合わせ、「その軸は本当に正しいのか」をフラットに議論する。

**なぜこれがAIにできないのか。**

理由は単純だ。面接は「リアルタイムの即興」だからだ。候補者の表情、声のトーン、言葉の選び方、沈黙の長さ。それらの微細なシグナルを読み取りながら、話の方向を変え、深掘りの角度を変え、アトラクトのタイミングを計る。これは対話であり、関係構築であり、信頼の醸成だ。

そして、この仕事の説得力は、私自身が共同創業者として経営を経験し、資金調達を主導し、トップセールスとして数百名の候補者と向き合ってきた「人生の厚み」から来ている。AIはフレームワークを模倣できるかもしれないが、**「この人の言葉には重みがある」と候補者に感じさせることはできない。**

## ここまでの結論 — 何が残り、何がなくなるのか

自分の仕事をAIで解体した結果、見えてきた構造はこうだ。

```
┌─────────────────────────────┐
│    自動化できる領域          │
│    ＝作業・集計・生成        │
│                             │
│  候補者登録、文面生成、      │
│  ダッシュボード、レポート    │
├─────────────────────────────┤
│    部分自動化の領域          │
│    ＝型はシステム、判断は人  │
│                             │
│  スカウト運用、求人票作成、  │
│  選考スピード管理            │
├─────────────────────────────┤
│    自動化できない領域        │
│    ＝対話・信頼・判断        │
│                             │
│  面接・クロージング          │
│  経営者のビジョン言語化      │
│  エージェントのファンづくり  │
│  ペルソナのペイン言語化      │
└─────────────────────────────┘
```

**上の層は、いずれAIに置き換わる。** むしろ、自分で積極的に自動化すべきだ。それができること自体が「オーケストレーター」としての能力の証明になる。

**下の層は、残る。** そしてここを磨くことでしか、AI時代に生き残ることはできない。

面白いのは、自動化を進めれば進めるほど、自動化できない部分の価値が上がるということだ。作業が消えれば、判断と対話だけが残る。RPOの提供価値が「採用に必要な作業をやってくれる人」から「採用の意思決定と候補者の口説きができる人」に純化される。

**AIは仕事を奪うのではなく、仕事の本質を露わにする。**

---

## 最後に — Last One Standing

このプロジェクトを「Last One Standing」と名付けた。

最後まで立っている者。AIがホワイトカラーの仕事を変えていく中で、自分は何で立ち続けるのかを明確にするための取り組みだ。

私の答えは、まだ途中だ。しかし、方向は見えた。

**自動化できるものは全て自動化する。** それは効率化であると同時に、自分のオーケストレーション能力の証明でもある。

**自動化できないものは、徹底的に磨く。** 経営者の言語化されていないビジョンを引き出す力。候補者のキャリアの本質に触れる対話力。エージェントを本気にさせる情熱。これらは、私の人生の経験からしか生まれない。

両方をやる。それが、Last One Standingであるための条件だと思っている。

本記事の続編では、具体的に自動化したシステムの詳細と、「自動化できない強み」をどう言語化してクライアントに訴求するかについて書く予定だ。
